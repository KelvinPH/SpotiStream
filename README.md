## üöÄ Quick Start (First-Time Setup)

1. Create a Spotify app ‚Üí <https://developer.spotify.com/dashboard/applications>  
2. **App Settings ‚Üí Redirect URIs ‚Üí Add** (exactly):  
   `https://kelvinph.github.io/SpotiStream/overlay.html` ‚Üí **Save**  
3. Copy your **Client ID**.  
4. Open the **Configurator**, design your overlay, and **Copy URL**.  
5. In **OBS** ‚Üí **Sources** ‚Üí **+** ‚Üí **Browser** ‚Üí paste the copied URL.  
   - A good starting size is **900 √ó 300** (you can resize later).  
6. Right-click the Browser Source ‚Üí **Interact** ‚Üí **Connect to Spotify** ‚Üí paste your **Client ID** ‚Üí approve.  
7. Start playing music in Spotify. The overlay updates automatically.

> **Tip:** Do the first login **inside OBS** (Interact). Tokens are stored in OBS‚Äôs browser cache.

## üß© OBS Setup

1. **Browser Source** ‚Üí paste the URL generated by the Configurator.  
2. **Scale** by resizing the source; the overlay is fully responsive.  
3. To keep tokens and avoid flicker, open Browser Source **Properties** and uncheck:  
   - **Refresh browser when scene becomes active**  
   - **Shutdown source when not visible**  
4. First-time auth: **Right-click ‚Üí Interact** inside OBS and connect to Spotify there.

## ‚ùì Troubleshooting

**Redirect / mismatch error**  
Ensure the Spotify app‚Äôs **Redirect URI** matches your overlay exactly:  
`https://kelvinph.github.io/SpotiStream/overlay.html`

**OBS asks to login again**  
Web browsers and OBS use different storage. Do the **first login in OBS** using **Interact**.

**Overlay shows ‚ÄúNothing playing‚Äù**  
Start playback on the same Spotify account that you authenticated. Free accounts work.

**Changes don‚Äôt appear**  
You may be seeing a cached file. Use **Ctrl+F5** (hard refresh) or open the overlay in a private/incognito window.

**Performance tips**  
- Keep **blur** modest on low-end systems.  
- Use **Bar** layout for very small overlays.
